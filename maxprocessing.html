# Connecting Processing and Max/MSP/Jitter We'll connect Processing and Max by using network messages, which are structured using the OSC protocol. The basic form of the OSC protocol sends messages as a pathname (a string with ```/``` slashes, e.g. ```/foo/bar```) followed by a list of arguments (which can be ints, floats and strings). First however we need to install an extension library for Processing to add OSC support. [Download the oscP5 library from here](http://www.sojamo.de/libraries/oscP5/). Unzip this file and copy the ```oscP5``` directory it contains into the ```libraries``` subdirectory of the ```Processing``` directory. (On OSX this is in ```Documents/Processing/libraries```, on Windows it is ...?) Max has enough objects built-in for us to get started with OSC without needing to download anything. ## Sending from Processing ```java // load the P5 libraries: import oscP5.*; import netP5.*; // create the objects we need: OscP5 oscP5; NetAddress myRemoteLocation; void setup() { // start oscP5: oscP5 = new OscP5(this, 11111); // set up the address we will send messages to: // "127.0.0.1" means 'send to the same machine this is running on' // 12345 is the port number Max is listening on myRemoteLocation = new NetAddress("127.0.0.1", 12345); } // send a message when the mouse is pressed: void mouseDragged() { // start building a message (with the message address /test): OscMessage myMessage = new OscMessage("/test"); // add some arguments: myMessage.add(mouseX); /* add an int to the osc message */ myMessage.add(1.23); /* add a float to the osc message */ myMessage.add("hello"); /* add a string to the osc message */ // and finally, send it: oscP5.send(myMessage, myRemoteLocation); } void draw() { // whatever you draw goes here } ``` > Possible problems: you can only have one instance of this running in Processing. If this happens, and error about "address already in use" will appear in the console. Just close the other sketches and run again. Sometimes oscP5 seems to get confused and complains even when other sketches are not running. If that happens, change the number in the OscP5 constructor to another five-digit value. > You might also need to open up these ports on the computer's firewall settings. ## Receiving in Max Just create a [udpreceive 12345] object. Whatever you send in Processing will come out of the Max object's outlet. You probably want to use [route] to divert messages according to the address, and then [unpack] to get the individual arguments, like the patcher below.
